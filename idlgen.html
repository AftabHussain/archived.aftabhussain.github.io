IDL Code Generation for Linux LCDs

  • Here we see the IDL (Interface Description Language) code that should be generated for different scenarios of module code within an LCD (Light-weight Capability Domain).

Scenario 1: Function calls passing structure variables.

  • Actual Code: 
    • err = register_netdev(netdev);
  • IDL: 
    • rpc int register_netdev(projection net_device_reg[alloc(callee) * dev])

    Scenario 2: Structure variable access inside a function definition.

    • Actual Code: 
      • static void dummy_setup (struct net_device * dev){
 dev -> flags |= IFF_NOARP;
}
    • IDL: 
      • projection  dummy_setup.dev {
unsigned int priv_flags;
}
rpc void dummy_setup (projection dummy_setup.dev * dev)
